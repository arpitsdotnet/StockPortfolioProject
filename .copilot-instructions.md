# Stock Portfolio Management System - Copilot Instructions

## 🎯 Core Standards

All code must follow these mandatory standards. Deviations require justification.

---

## 📋 API & Architecture

**Controllers:** Use `ControllerBase` in `StockPortfolio.API/Controllers/`. Apply `[ApiController]` and `[Route("api/[controller]")]`. Return appropriate HTTP status codes (200, 201, 204, 400, 404, 409, 500).

**Handlers & Services:** Use handler pattern (e.g., `CreateSharePriceHandler`) for business logic. Handlers accept request DTO → return `Result<T>`. Inject `ApplicationDbContext` directly—**no repository pattern**. Use LINQ with `AsNoTracking()` for read-only queries.

---

## 💉 Dependency Injection

- **Always** use constructor-based DI
- Register all dependencies in `Program.cs`
- Lifetimes: `Scoped` (DbContext, services), `Transient` (stateless), `Singleton` (shared/thread-safe)

---

## 📝 Code Organization

**File Structure:** One class/interface/DTO per file. Feature-based organization:
```
Features/SharePrices/
├── CreateSharePrice/
│   ├── CreateSharePriceRequest.cs
│   └── CreateSharePriceHandler.cs
├── DeleteSharePrice/
```

**Naming Conventions:**
| Element | Convention | Example |
|---------|-----------|---------|
| Classes | PascalCase | `CreateSharePriceHandler` |
| Interfaces | I + PascalCase | `ISharePriceService` |
| Private fields | _camelCase | `_context` |
| Local vars | camelCase | `userId` |
| Constants | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT` |

---

## 🎁 DTOs & Result Pattern

**Use DTOs for:** API payloads, cross-layer communication, decoupling entities from contracts. Create separate Request and Response DTOs.

**Result<T> Pattern:**
```csharp
// Success: Result<SharePriceDto>.Success(dto)
// Failure: Result<SharePriceDto>.Failure(new Error(...))
```

---

## 💬 Comments

- Add **1-2 liner comments** for complex logic only
- Use XML docs (`///`) on public members
- Code should be self-documenting

---

## ✅ Best Practices

**General:** Use `async/await` for I/O. Always pass `CancellationToken`. Use `??` and `?.` operators. Validate input at controller/handler level. Keep methods small (SRP).

**Error Handling:** Return `Result<T>` for operations that can fail. Map errors to HTTP status codes. Log errors with `ILogger`. Never expose internal exceptions.

**Performance:** Use `AsNoTracking()` for read-only queries. Implement pagination for large sets. Use `Include()` to prevent N+1 queries.

**Testing:** Write unit tests for handlers/business logic. Use AAA pattern. Mock external dependencies.

## 🔗 Key Integration Points

**Handler Example:**
```csharp
public class CreateSharePriceHandler
{
    private readonly ApplicationDbContext _context;

    public CreateSharePriceHandler(ApplicationDbContext context) => _context = context;

    public async Task<Result<SharePriceDto>> Handle(CreateSharePriceRequest request, CancellationToken ct)
    {
        if (request?.Price <= 0)
            return Result<SharePriceDto>.Failure(
                new Error(ErrorType.VALIDATION, ErrorCode.BAD_REQUEST, "Invalid price"));

        // Business logic
        var entity = new SharePrice { Price = request.Price, Date = request.Date };
        _context.SharePrices.Add(entity);
        await _context.SaveChangesAsync(ct);

        return Result<SharePriceDto>.Success(new SharePriceDto { /* map */ });
    }
}
```

**Controller Example:**
```csharp
[ApiController]
[Route("api/[controller]")]
public class SharePricesController : ControllerBase
{
    private readonly CreateSharePriceHandler _handler;

    public SharePricesController(CreateSharePriceHandler handler) => _handler = handler;

    [HttpPost]
    public async Task<IActionResult> Create(CreateSharePriceRequest request, CancellationToken ct)
    {
        var result = await _handler.Handle(request, ct);
        return result.IsSuccess 
            ? Created("", new ResultDto { IsSuccess = true, Value = result.Value })
            : result.Error.Code == ErrorCode.BAD_REQUEST 
                ? BadRequest(new ResultDto { IsSuccess = false, Error = result.Error })
                : StatusCode(500, new ResultDto { IsSuccess = false, Error = result.Error });
    }
}
```
